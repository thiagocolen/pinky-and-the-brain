# Documentation Content Plan

This document plans the Docusaurus documentation website scaffolded in `docs/` for the
`pinky-and-the-brain` agent service, following the pattern established in
`thiagocolen/patb-cli#1`. It replaces the default `create-docusaurus` placeholder
content with real material derived from `src/`, `readme.md`, and `CHANGELOG.md`,
organized into an End-User section and a Developer & Technical section.

---

### 1. Source Material

- `readme.md`: architecture overview, directory structure, setup/installation, the four
  entrypoints (ACP stdin/stdout, REPL CLI, REST/SSE server, MCP server), `patb-cli`
  integration, Zed editor integration.
- `CHANGELOG.md`: v0.2.0 feature list (LangGraph orchestration, persistence, REST/SSE,
  MCP, Terraform infrastructure).
- `src/index.ts`, `src/cli.ts`, `src/server.ts`, `src/mcp.ts`: the four entrypoints.
- `src/config.ts`: environment variables, Zod schema, `.env` loading order.
- `src/agents/graph.ts`, `the-brain.ts`, `specialists.ts`, `types.ts`: the LangGraph
  supervisor/specialist orchestration, routing, and shared state shape.
- `src/protocol/acp-server.ts`, `messages.ts`: the ACP JSON-RPC methods and schemas.
- `src/storage/sqlite.ts`, `s3.ts`: local checkpointing and cloud state persistence.
- `src/utils/model.ts`: LLM provider selection (Gemini vs OpenAI).
- `terraform/*.tf`: AWS infrastructure (ECR, ECS, CloudFront, IAM, SSM parameters).

All facts used in the docs come directly from these files - no invented behavior.

---

### 2. Information Architecture

The Docusaurus sidebar is auto-generated from the `docs/docs/` folder tree
(`sidebars.ts` -> `{type: 'autogenerated', dirName: '.'}`), so the IA below is expressed
purely as a folder structure using `_category_.json` and `sidebar_position` frontmatter.

```
docs/docs/
  intro.mdx                              (replaces default intro, position 1)
  end-user/                              ("End-User Documentation", position 2)
    installation.mdx                     (prereqs, npm install, npm run build, .env setup)
    configuration.mdx                    (all env vars, .env loading order, API key masking)
    cli-usage.mdx                        (REPL walkthrough: npm run cli)
    server-usage.mdx                     (REST/SSE server: npm run server, endpoints, curl examples)
    mcp-usage.mdx                        (MCP server: npm run mcp, run_agent tool)
    acp-usage.mdx                        (raw ACP stdin/stdout server + Zed editor integration
                                           + patb-cli as the recommended remote client)
    command-reference.mdx                (npm scripts table + REST endpoint table)
    faq.mdx                              (Q&A knowledge base)
    troubleshooting.mdx                  (common setup/runtime errors, expanded)
  developer/                              ("Developer & Technical Documentation", position 3)
    project-structure.mdx                (repo layout table, derived from readme.md)
    architecture.mdx                     (agent graph, routing, persistence, data-flow diagrams)
    source-code-reference.mdx            (per-module reference: agents/protocol/storage/utils)
    acp-protocol.mdx                     (JSON-RPC methods & error codes implemented)
    infrastructure.mdx                   (Terraform/AWS/Docker deployment topology)
    contributing.mdx                     (build/test/lint loop, docs dev loop, conventions)
```

**End-User Documentation** covers user manuals (installation, configuration), how-to
guides/tutorials (CLI/server/MCP/ACP usage), and an FAQ / knowledge base (FAQ +
troubleshooting).

**Developer & Technical Documentation** covers source code documentation
(source-code-reference, acp-protocol), project structure, and architecture design
(architecture, infrastructure).

### 3. Cleanup of placeholder content

- Remove `docs/docs/tutorial-basics/` and `docs/docs/tutorial-extras/` (default
  Docusaurus tutorial placeholders).
- Trim `docs/blog/` sample posts down to a single real welcome post plus real
  `authors.yml`/`tags.yml` metadata, removing the generic Docusaurus sample posts.
- Update `docs/src/pages/index.tsx` hero button/label to point at the real docs instead
  of the "Docusaurus Tutorial" link.
- Replace the 3 generic features in `docs/src/components/HomepageFeatures/index.tsx`
  with 3 project features (Multi-Agent LangGraph Orchestration, Multiple Protocol
  Entrypoints, Cloud-Native Persistence), reusing the already-bundled `undraw_*` SVGs
  (no new static assets needed).

---

### 4. Verification

- `npm run docs:build` (`npm run build --prefix docs`) from the repo root, to confirm
  the site builds cleanly under `onBrokenLinks: 'throw'`.
- Manual check of the generated sidebar categories/order.

---

### 5. Delivery

- Branch: `open-swe/docs-website`.
- Single PR against `master`, title referencing this issue, following the repository's
  `Description` / `Release Note` / `Test Plan` body convention.
- Publishing to GitHub Pages (CI workflow, `gh-pages` branch, repository Pages settings)
  is intentionally left out of this change and can be added in a follow-up, mirroring
  the two-step approach used in `patb-cli#1` (plan/content first, publish automation
  second).
